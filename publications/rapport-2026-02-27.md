

# Claude Code Scout — weekrapport 27 februari 2026

## Samenvatting

Deze week wordt gedomineerd door drie grote ontwikkelingen. **Ten eerste:** Anthropic heeft **automatische sessiegeheugen (MEMORY.md)** uitgerold — Claude Code onthoudt nu zelfstandig projectpatronen, voorkeuren en fix-strategieën tussen sessies. Dit is een fundamentele verandering die je hele persistentie-strategie beïnvloedt. **Ten tweede:** Check Point Research publiceerde een **kritieke kwetsbaarheid (CVE-2025-59536)** waarbij kwaadaardige repositories via hooks en configuratiebestanden willekeurige code kunnen uitvoeren op je machine — directe actie vereist. **Ten derde:** de opkomst van **Agent Teams** maakt het mogelijk om meerdere Claude Code-instanties gecoördineerd op een gedeeld project te laten samenwerken, wat een verschuiving betekent van lineaire naar parallelle development workflows.

Daarnaast zijn er waardevolle inzichten rond kostenbesparing (progressive disclosure bespaart ~82% tokens), context management (/compact reduceert 70K tokens tot ~4K), en Claude Code Security als nieuwe vulnerability scanner die 500+ bugs vond in beproefde open-source codebases.

### Geheugen & persistentie

**Voorstel:** Activeer en configureer het nieuwe automatische sessiegeheugen (MEMORY.md)
**Bron:** The Decoder — [Claude Code now remembers your fixes, your preferences, and your project quirks on its own](https://the-decoder.com/claude-code-now-remembers-your-fixes-your-preferences-and-your-project-quirks-on-its-own/)
**Type:** fundamenteel
**Toelichting:** Claude Code maakt nu automatisch een MEMORY.md-bestand per project aan in `~/.claude/projects/<project>/memory/`. Het slaat zelfstandig projectpatronen, debugfix-technieken, architectuurconventies en ontwikkelaarsvoorkeuren op en haalt deze automatisch op in latere sessies. Dit elimineert de noodzaak om elke sessie handmatig context te herstellen. Aangezien je huidige setup geen persistentie-strategie beschrijft, is dit een directe verbetering. Controleer of deze feature actief is en overweeg de initiële MEMORY.md te cureren zodat Claude met de juiste basiskennis start.

```yaml
# Controleer of session memory actief is
# Locatie: ~/.claude/projects/<project-naam>/memory/
# Na een sessie zou hier automatisch een MEMORY.md moeten verschijnen

# Je kunt handmatig initiële observaties toevoegen:
# ~/.claude/projects/<jouw-project>/memory/MEMORY.md
```

### Beveiliging

**Voorstel:** Audit je hooks en MCP-configuratie op supply chain-risico's — CVE-2025-59536
**Bron:** Check Point Research — [Caught in the Hook: RCE and API Token Exfiltration Through Claude Code Project Files](https://research.checkpoint.com/2026/rce-and-api-token-exfiltration-through-claude-code-project-files-cve-2025-59536/)
**Type:** fundamenteel
**Toelichting:** Check Point Research onthulde dat kwaadaardige repositories via `.claude/`-configuratiebestanden willekeurige shell-commando's kunnen uitvoeren en Anthropic API-keys kunnen exfiltreren op de machines van alle medewerkers, zonder expliciete toestemming. Eén schadelijke commit in een gedeeld project kan elke ontwikkelaar compromitteren. Dit betreft een supply chain-aanval die hooks, MCP-servers en environment variables misbruikt. **Directe actie:** review alle repository-level configuratie, kloon geen onvertrouwde repositories met Claude Code actief, en overweeg sandboxing in te schakelen.

```yaml
# Beveiligingsmaatregelen:
# 1. Review .claude/ directories in alle repositories op onbekende hooks/configs
# 2. Gebruik nooit --dangerously-skip-permissions op onvertrouwde repos
# 3. Activeer sandboxing voor OS-level enforcement:
#    claude config set sandbox true
# 4. Controleer of MCP-servers alleen via vertrouwde bronnen draaien
```

### Hooks

**Voorstel:** Implementeer hooks als afdwingbare regels in plaats van richtlijnen
**Bron:** Pixelmojo — [Claude Code Hooks Guide: All 12 Lifecycle Events Explained](https://www.pixelmojo.io/blogs/claude-code-hooks-production-quality-ci-cd-patterns)
**Type:** fundamenteel
**Toelichting:** Hooks transformeren best-practice richtlijnen (die Claude kan negeren) in deterministisch afdwingbare regels die altijd uitvoeren op specifieke lifecycle-punten. Dit is fundamenteel anders dan instructies in CLAUDE.md: hooks draaien als shell-commando's en zijn niet afhankelijk van Claude's interpretatie. Aangezien je geen hooks beschrijft in je huidige setup, is dit een kans om kwaliteitscontroles (linting, formatting, testuitvoering) te automatiseren.

```yaml
# Voorbeeld hooks in .claude/hooks.json:
{
  "PreToolUse": [
    {
      "match": "Write|Edit",
      "command": "echo 'Controleer of tests bestaan voor gewijzigde bestanden'"
    }
  ],
  "PostToolUse": [
    {
      "match": "Write",
      "command": "npx eslint --fix $CLAUDE_FILE_PATH"
    }
  ]
}
```

**Let op:** Gezien de CVE-2025-59536 kwetsbaarheid, review altijd hooks in gekloonde repositories en gebruik alleen hooks die je zelf hebt geschreven of vertrouwt.

## Paradigma-check

### 1. Van geheugen-als-configuratie naar geheugen-als-emergentie

De introductie van automatisch sessiegeheugen (MEMORY.md) verschuift het paradigma fundamenteel: waar je eerder alle context **vooraf** moest configureren in CLAUDE.md, leert Claude nu **zelfstandig** van je patronen. Dit stelt de aanname ter discussie dat je alles expliciet moet specificeren. De nieuwe strategie is: gebruik CLAUDE.md voor regels en intenties, en laat MEMORY.md organisch groeien met projectkennis. Dit vereist periodiek cureren — niet alles wat Claude "onthoudt" is waardevol.

### 2. Hooks zijn een aanvalsvector, niet alleen een productiviteitstool

De CVE-2025-59536-onthulling toont dat het hele hooks-systeem — door velen gepresenteerd als dé manier om workflows te automatiseren — tegelijkertijd een kritiek aanvalsoppervlak is. Repository-gecontroleerde configuratiebestanden kunnen hooks uitvoeren zonder expliciete toestemming. Dit stelt de gangbare aanname ter discussie dat meer automatisering automatisch beter is. **De nieuwe vuistregel:** behandel elke `.claude/`-directory in een gekloonde repository als potentieel kwaadaardig, net zoals je `.env`-bestanden met argwaan benadert.

### 3. De "productiviteitspanik" als contra-signaal

Bloomberg rapporteert dat AI coding agents een "productiviteitspanik" veroorzaken in tech-bedrijven: de verwachting dat engineers meer moeten produceren leidt tot overwerk in plaats van betere output. Dit is een belangrijk contra-signaal voor iedereen die Claude Code primair als productiviteitsversneller positioneert. De werkelijke waarde zit mogelijk niet in snelheid maar in **kwaliteitsverhoging** — betere tests, veiliger code, minder regressies. Overweeg of je eigen doelstellingen meer op velocity of op quality gericht moeten zijn.
